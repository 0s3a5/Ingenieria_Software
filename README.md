Documentación Técnica: Módulo de Mapa y Ubicación1. Descripción GeneralEsta sección de la aplicación es responsable de mostrar un mapa interactivo utilizando Google Maps. Su función principal es solicitar los permisos necesarios al usuario para obtener su ubicación GPS en tiempo real, centrar la cámara en su posición actual y mostrar una serie de puntos de interés (pines) utilizando datos locales de prueba (Mock Data). Al presionar estos pines, se despliega una ventana de información personalizada (InfoWindow) con detalles específicos del lugar.2. Dependencias y LibreríasPara que este módulo funcione correctamente, el proyecto utiliza las siguientes implementaciones en el archivo build.gradle (Module :app):Play Services Location: com.google.android.gms:play-services-location:21.0.1 (Utilizada para acceder al hardware GPS del dispositivo mediante FusedLocationProviderClient).Google Maps SDK: (Se asume preconfigurada en el proyecto para renderizar el SupportMapFragment).3. Permisos del SistemaLa aplicación requiere permisos de ubicación explícitos declarados en el AndroidManifest.xml para funcionar correctamente.android.permission.ACCESS_FINE_LOCATION: Para obtener la ubicación exacta mediante GPS.android.permission.ACCESS_COARSE_LOCATION: Para obtener una ubicación aproximada mediante redes móviles o Wi-Fi.El manejo de permisos en tiempo de ejecución se realiza mediante ActivityResultContracts.RequestPermission().4. Estructura de Datos (Modelo)Para el manejo de la información de los pines en un entorno de desarrollo (local), se utiliza la siguiente estructura de datos data class:AtributoTipo de DatoDescripciónnombreStringEl nombre comercial o título del punto de interés.direccionStringLa dirección física del lugar.precioOfertaStringTexto que representa un valor o precio de ejemplo.ubicacionLatLngObjeto de Google Maps que contiene la Latitud y Longitud exacta.5. Archivos PrincipalesMapaFragment.ktEs el controlador principal de la vista del mapa. Sus responsabilidades incluyen:Inicializar el mapa asíncronamente mediante OnMapReadyCallback.Instanciar FusedLocationProviderClient para leer la última ubicación conocida del usuario.Solicitar dinámicamente los permisos de ubicación si el usuario no los ha otorgado.Cargar la lista local de tiendas (TiendaLocal) y asignarlas como el atributo tag de cada marcador (Marker).Sobrescribir el adaptador nativo del mapa (setInfoWindowAdapter) para inyectar el diseño personalizado.custom_info_window.xmlEs el diseño visual (UI) del pop-up que aparece al tocar un pin. Está estructurado como un LinearLayout vertical que contiene:Un TextView para el Título (en negrita).Un TextView para la Dirección (color gris oscuro).Un TextView para el Precio (color verde para resaltar).
